# Makimono

Makimono is a personal backlog PWA for tracking series, movies, games, books, music, and podcasts. The current codebase is a TanStack Start + React + TypeScript app with Tailwind CSS v4, shadcn-compatible primitives, ES/EN localization, a branded home/auth experience, dashboard display modes, drawer-based add/edit flows, archive/settings screens, and an optional PocketBase-backed auth + repository mode.

## Quick path

1. Install dependencies:
   ```bash
   npx pnpm install
   ```
2. Start the app:
   ```bash
   npx pnpm dev
   ```
3. Verify before review:
   ```bash
   npx pnpm typecheck
   npx pnpm test
   npx pnpm build
   ```

## What this project is

| Area | Decision |
|---|---|
| Product goal | A bilingual mock UI for managing a personal content backlog. |
| Runtime stack | TanStack Start, React 19, TypeScript 6, Vite 8. |
| Styling | Tailwind CSS v4 tokens plus shadcn-compatible CSS variables. |
| UI primitives | Local primitives under `src/components/ui/`. |
| Data model | `InterestRepository` with browser-local localStorage fallback, plus an optional PocketBase-backed implementation for authenticated users. |
| Localization | ES/EN dictionaries under `src/i18n/`. |
| Testing | Vitest + Testing Library, focused on visible behavior. |
| Scope boundary | Recommendation/AI improvements and real Gemini integration remain out of scope. Historical MVP slices were mock-first; this repository snapshot also includes an optional PocketBase auth + interests persistence setup. |

## Main user flows

| Flow | Purpose |
|---|---|
| Home | Branded Makimono entry point with animated logo, version, working-on list, and PocketBase sign-in/register card when auth is enabled. |
| Dashboard | Main backlog surface with category filters, search, display modes, dynamic logo/favicon, status actions, and add/edit drawers. |
| Add/Edit interest | Drawer-based item form with category details, tags, notes, optional cover lookup, save, and delete actions. |
| Archive | Full-screen archive for completed/deleted items with restore actions and search. |
| Settings | Full-screen settings for language, dashboard display preference, app version, and logout. |

Route notes:

- `/` and `/dashboard` are the primary entry points.
- Add/edit are exposed as dashboard interactions and still have direct route fallbacks for deep links/tests.
- The mock suggester route is preserved in code but hidden from the current dashboard UI.

## Execution steps

### 1. Prerequisites

- Node.js compatible with the checked lockfile.
- `pnpm` can be run through `npx pnpm` in this environment.
- Do not rely on global `pnpm`; it may not be available on `PATH`.

### 2. Install

```bash
npx pnpm install
```

This uses `pnpm-lock.yaml` and installs the TanStack Start, React, Tailwind, Radix, and test dependencies.

### 3. Run locally

```bash
npx pnpm dev
```

Then open the local Vite/TanStack Start URL printed by the command.

### 4. Build for production

```bash
npx pnpm build
```

This generates client and server build output under ignored build directories.

### 5. Run the production build locally

```bash
npx pnpm start
```

Use this only after `npx pnpm build` has completed successfully. TanStack Start is served from Nitro's `.output/server/index.mjs` output, not from a static Vite `dist` preview.

## Important implementation notes

- Local mode still works when `VITE_POCKETBASE_URL` is not configured. When it is configured, dashboard routes require PocketBase auth and use the remote `interests` collection instead of localStorage.
- `src/routeTree.gen.ts` is generated by TanStack Router tooling. Do not hand-edit it except through route generation/build tooling.
- Dialog and Sheet primitives require callers to pass localized `closeLabel` values; Drawer intentionally does not render a built-in close button.
- Radix ToggleGroup filter pills expose radio semantics; tests should query them as radios when relevant.
- The home/dashboard visual identity uses public PWA/logo assets under `public/`; do not stage local design-source files unless explicitly approved.
- Local AI/tooling workspace files are ignored: `.atl/`, `.gga`, `.mcp.json`, `opencode.json`, and `untitled.op`.

## PocketBase setup

1. Install dependencies if you have not already:

   ```bash
   npx pnpm install
   ```

2. Set the public PocketBase URL in your local environment file:

   ```bash
   VITE_POCKETBASE_URL=https://pb.soyun.ninja
   ```

3. Import `docs/pocketbase-collections.json` into PocketBase so the `interests` collection exists with the expected auth rules.

4. Start the app normally with `npx pnpm dev`.

Behavior notes:

- With `VITE_POCKETBASE_URL` set, protected dashboard surfaces show a login/register gate until the user authenticates.
- After login, item CRUD uses PocketBase and keeps the SDK session persisted through `authStore`.
- Without `VITE_POCKETBASE_URL`, the app stays in localStorage mode for development and existing local tests.

## Optional cover metadata

- Optional cover metadata is documented as a separate approved slice under `openspec/changes/add-cover-metadata-cache/`.
- Current provider coverage is TMDB for series/movies, RAWG for games, Open Library for books, and MusicBrainz + Cover Art Archive for music/podcasts.
- If that slice is unavailable or disabled in a local environment, add/edit flows still save normally and cards fall back to category color only.
