# MeInteresa

MeInteresa is a mock-only personal backlog UI for tracking links, books, movies, series, and games. The current baseline is a TanStack Start + React + TypeScript app with Tailwind CSS v4, shadcn-compatible primitives, local mock data, ES/EN localization, dashboard flows, an adaptive add form, archive view, and a mock suggester.

## Quick path

1. Install dependencies:
   ```bash
   npx pnpm install
   ```
2. Start the app:
   ```bash
   npx pnpm dev
   ```
3. Verify before review:
   ```bash
   npx pnpm typecheck
   npx pnpm test
   npx pnpm build
   ```

## What this project is

| Area | Decision |
|---|---|
| Product goal | A bilingual mock UI for managing a personal content backlog. |
| Runtime stack | TanStack Start, React 19, TypeScript 6, Vite 8. |
| Styling | Tailwind CSS v4 tokens plus shadcn-compatible CSS variables. |
| UI primitives | Local primitives under `src/components/ui/`. |
| Data model | Local mock repository only. No persistence or backend calls. |
| Localization | ES/EN dictionaries under `src/i18n/`. |
| Testing | Vitest + Testing Library, focused on visible behavior. |
| Scope boundary | No authentication, database, backend API, or real Gemini integration in this MVP. |

## Main user flows

| Route | Purpose |
|---|---|
| `/` | Foundation landing screen and entry point. |
| `/dashboard` | Backlog dashboard with category filters and status actions. |
| `/dashboard/add` | Adaptive add flow using desktop Dialog and mobile Sheet behavior. |
| `/dashboard/suggest` | Mock suggester returning exactly 3 recommendations with reason + CTA. |
| `/dashboard/archive` | Archive fallback screen for completed items. |

## Execution steps

### 1. Prerequisites

- Node.js compatible with the checked lockfile.
- `pnpm` can be run through `npx pnpm` in this environment.
- Do not rely on global `pnpm`; it may not be available on `PATH`.

### 2. Install

```bash
npx pnpm install
```

This uses `pnpm-lock.yaml` and installs the TanStack Start, React, Tailwind, Radix, and test dependencies.

### 3. Run locally

```bash
npx pnpm dev
```

Then open the local Vite/TanStack Start URL printed by the command.

### 4. Build for production

```bash
npx pnpm build
```

This generates client and server build output under ignored build directories.

### 5. Preview the production build

```bash
npx pnpm preview
```

Use this only after `npx pnpm build` has completed successfully.

## Important implementation notes

- The MVP is intentionally mock-only. Keep persistence, authentication, backend calls, and real Gemini integration out unless a later spec adds them.
- `src/routeTree.gen.ts` is generated by TanStack Router tooling. Do not hand-edit it except through route generation/build tooling.
- Dialog and Sheet primitives require callers to pass localized `closeLabel` values.
- Radix ToggleGroup filter pills expose radio semantics; tests should query them as radios when relevant.
- Local AI/tooling workspace files are ignored: `.atl/`, `.gga`, `.mcp.json`, `opencode.json`, and `untitled.op`.
